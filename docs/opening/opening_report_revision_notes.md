# 开题报告修订说明

## 依据

本次修订参照以下材料：

- `docs/opening/source_requirements/opening_notice_2025.pdf`
- `docs/opening/source_requirements/opening_report_template_0604.docx`
- `academic-research-suite` 中 academic-paper workflow 的结构、引用和格式要求
- `humanizer` 的文字清理规则

## 结构调整

报告已按学校模板要求组织为以下栏目：

- 课题背景、目的和意义
- 国内外研究现状
- 研究目标与研究内容
- 研究方案与可行性分析
- 进度安排
- 预期成果
- 预期创新点
- 精读文献清单
- 主要参考文献
- 签字页

## 学校通知要求对照

- 开题报告内容覆盖文献综述、选题意义、研究内容、研究方案、预期目标和可行性分析。
- 参考文献列出 40 篇，满足硕士文献阅读量不少于 40 篇的形式要求。
- 单独列出 15 篇精读文献清单，便于答辩时抽问。
- 文献中英文文献占比较高，满足外文文献不少于三分之一的要求。

## 学术规范化处理

- 将实验结果表述为开题阶段可行性证据，避免写成完整系统性能。
- 明确 Spider smoke test 不是完整 Spider benchmark 跑分。
- 明确 Skill Router v3 的 13/13 来自已知 SQL+ 失败样例。
- 增加“已有工作局限”，说明数据规模、SQL+ 语法覆盖、非 gold 反馈修正和达梦方言适配的边界。

## 去 AI 化处理

- 减少“主要体现在三个方面”等模板化段落。
- 减少泛化意义拔高，改为围绕具体研究假设展开。
- 减少“显著、明显、关键”等无证据强化词。
- 将部分清单式结论改为更自然的因果说明。
- 保留学术报告需要的正式语气，不加入口语化表达。

## 仍需人工补充

- 学号、姓名、专业、导师姓名。
- 导师要求的最终题目表述。
- 若学校要求正式 Word 模板版式，需要在 WPS/Word 中检查页边距、字体、行距和签字页。
- 参考文献条目建议在定稿前进一步补 DOI、页码或会议/期刊完整信息。
